# Spec: 格式工坊 MVP

## Objective

构建一个面向普通用户的中文在线格式转换网站。用户无需登录，文件始终在浏览器本地处理，不上传到服务器。

## User Flows

1. 用户拖放、选择文件或选择文件夹导入最多 20 张 JPG、PNG、WebP 图片，统一选择输出格式、质量和最大尺寸，然后逐个下载结果或打包下载。
2. 用户可以输出为与源文件相同的图片格式，用于压缩或等比缩放。
3. 用户选择一个 MP4、MP3、WAV、M4A、AAC、FLAC 或 OGG 文件，输出为 MP3、WAV、M4A、AAC、FLAC 或 OGG，并设置码率、采样率和声道。
4. 用户可以看到文件队列、整体进度、成功结果、部分失败提示、预览和下载入口。
5. 用户可以取消正在运行的任务、只重试失败文件，并将多个成功结果打包为 ZIP 下载。
6. 用户可以查看当前浏览器中最近 10 条转换记录，重新下载、逐条删除或全部清空。
7. 用户可以将 JPG、PNG、WebP 图片按队列顺序生成 PDF，并选择适应图片或 A4 页面。
8. 用户可以把 PDF 全部页面转为 JPG/PNG、合并多个 PDF，或按页码提取和逐页拆分 PDF。
9. 用户可以通过缩略图拖拽或按钮重新排序 PDF 页面，并旋转、复制、删除页面后重新导出。
10. 用户可以选择保留文本、标准压缩或强力压缩，并按需添加页码和中文文字水印。
11. 用户选择一个 MP4 后，可以按质量、最大分辨率、帧率和是否保留音频生成压缩后的 MP4。
12. 用户可以填写开始和结束秒数，精确裁剪 MP4，并预览、下载生成的视频片段。
13. 用户可以截取最长 30 秒的视频片段，选择 480/640/800 px 宽度和 8/12/15 fps 后生成 GIF。
14. 用户可以定位到视频中的任意有效时间点，并导出 JPG 或 PNG 单帧截图。
15. 用户可以按 90° 步进旋转视频，按 1:1、16:9、9:16 或 4:3 居中裁切画面，也可单独生成静音视频。
16. 用户可以从 MP4 提取第一条可用音轨并输出 MP3、M4A 或 WAV。
17. 用户可以在当前浏览器保存、应用和删除最多 8 条视频参数预设，预设不包含文件或源视频时间点。
18. 视频处理成功后，用户可以并排查看源视频与视频、截图、GIF 或音频结果。
19. 用户可以裁剪音频、调整音量、标准化响度、改变播放速度，并添加最长 10 秒的淡入淡出。
20. 用户可以旋转或翻转图片、添加文字水印，并为批量结果生成安全的连续文件名。
21. 用户选择文件后，可以查看尺寸或时长、MIME 类型、文件大小和修改时间。
22. 用户可以将网站安装为 PWA；首次在线加载相关工具后，可离线重新打开应用并使用已缓存资源。

## Project Structure

- `src/conversion/`: 文件验证、图片编码与 FFmpeg 音频格式转换
- `src/ui/`: DOM 渲染、批量队列和交互状态
- `src/history/`: IndexedDB 本地转换记录与保留策略
- `src/pdf/`: PDF 创建、合并、拆分、页数读取和本地页面渲染
- `src/video/`: MP4 校验、基础与高级视频命令、本地预设模型和 FFmpeg 视频运行器
- `src/ui/pdf-page-manager*`: PDF 缩略图编辑状态、页面管理界面和响应式样式
- `src/ui/video-workbench*`: 独立懒加载的视频工具箱界面与响应式样式
- `tests/`: 转换参数、格式边界和 UI 状态单元测试
- `public/ffmpeg/`: 构建前复制的本地 FFmpeg 核心资源
- `docs/`: 产品规格
- `tasks/`: 实现计划和执行清单

## Product Rules

### Image conversion

- 支持 JPG、PNG、WebP 输入和输出。
- 单张图片最大 25 MB。
- 每批最多 20 张，总大小最大 100 MB。
- 批量任务顺序执行，避免同时解码多张大图。
- JPEG 和 WebP 质量范围为 40%-100%；PNG 使用无损编码。
- 最大宽高均可留空。填写后图片在该边界内等比缩小，不放大小图。
- 单张图片解码后不得超过 4000 万像素。
- 图片可按 0/90/180/270° 旋转，并可独立水平或垂直翻转；旋转 90/270° 时交换输出宽高。
- 文字水印最多 40 个字符，支持五个受控位置和 10%-100% 不透明度，不解析 HTML。
- 批量重命名会清理路径分隔符、控制字符和危险前导字符，并自动追加至少两位序号。

### Audio conversion

- 支持 MP4、MP3、WAV、M4A、AAC、FLAC 和 OGG 输入。
- 支持 MP3、WAV、M4A、AAC、FLAC 和 OGG 输出，不生成视频 MP4。
- 每次仅处理一个媒体文件，最大 200 MB。
- 只提取第一条可用音轨，不输出视频轨。
- MP3、M4A、AAC 和 OGG 支持 128、192、256 和 320 kbps；WAV 和 FLAC 为无损输出，不使用码率设置。
- 采样率可保留原始值或设置为 44.1/48 kHz；声道可保留原始值或设置为单/双声道。
- OGG 使用 48 kHz 兼容采样率；原始或单声道 Vorbis 最高使用 192 kbps，只有明确选择双声道才开放更高码率。
- FFmpeg 临时输入保留经过验证的容器扩展名，便于正确识别解复用器。
- FFmpeg 核心资源由当前站点本地提供。
- 音频裁剪使用 0-43200 秒的有效时间范围，结束时间必须晚于开始时间且不得超过已读取的源时长。
- 音量范围为 0%-200%；响度标准化使用固定的 EBU R128 参数；播放速度仅允许 0.5/0.75/1/1.25/1.5/2 倍。
- 淡入和淡出分别限制为 0-10 秒；淡出起点按裁剪后、变速后的实际输出时长计算。

### Local file information and PWA

- 文件信息只从浏览器本地 `File`、图片和媒体元数据读取，不上传或持久化原文件。
- Service Worker 只处理同源 GET 请求，缓存应用外壳、构建资源和本地 FFmpeg/PDF 运行资源，不代理跨域请求。
- 安装按钮只在浏览器触发 `beforeinstallprompt` 后显示；开发环境不注册 Service Worker。
- 新版本激活时清理旧版应用缓存，在线导航优先获取最新页面，断网时回退到已缓存应用外壳。

### PDF toolbox

- 图片转 PDF 支持 JPG、PNG、WebP，每批最多 20 张、总计 100 MB，单张不超过 25 MB。
- 图片转 PDF 可按原图像素适配页面，或使用自动横竖方向的 A4 页面和 0-72 pt 边距。
- PDF 转图片支持 JPG 和 PNG，可选择标准、清晰或高清渲染倍率；多页结果可打包为 ZIP。
- 合并 PDF 每次支持 2-10 个文件、总计不超过 200 MB，可在生成前调整文件顺序。
- 拆分 PDF 支持 `1-3,5,8` 形式的页码，可合成一个新 PDF 或每页生成一个 PDF。
- 单个 PDF 最大 100 MB；单个输入或合并后的文档最多 500 页。
- 加密或设置打开密码的 PDF 暂不支持，并显示统一的可恢复错误提示。
- PDF 引擎首次进入工具箱时懒加载；PDF.js Worker 由当前站点本地提供。
- 页面管理一次接受 1 个不超过 60 MB、最多 100 页的 PDF；复制后输出最多 200 页。
- 页面缩略图支持鼠标拖拽排序，同时提供可键盘访问的前移、后移、旋转、复制和删除按钮。
- 保留文本模式复制原始页面结构；标准和强力压缩将页面渲染为 JPEG，文字选择、链接和表单会被扁平化。
- 页面图像压缩只有在结果更小时才采用，否则自动回退到体积更小的结构保留版本。
- 页码使用文档总页数格式；文字水印最多 40 个字符，并在浏览器 Canvas 中生成透明 PNG 后嵌入 PDF。

### Video toolbox

- 仅支持 MP4 输入，每次一个文件，最大 150 MB，最长 10 分钟。
- 文件进入 FFmpeg 前同时校验 `video/mp4` MIME 和 MP4 `ftyp` 容器签名。
- MP4 压缩提供更小体积、均衡和高清三档质量，可保留原始分辨率或限制为 1080p/720p/480p，可保留原始帧率或设置为 30/24 fps。
- 压缩和裁剪默认保留音轨并编码为 AAC，也可选择移除音频。
- 时间裁剪要求开始时间不小于零、结束时间晚于开始时间且不超过源视频时长。
- 视频转 GIF 的片段最长 30 秒，宽度限制为 480/640/800 px，帧率限制为 8/12/15 fps。
- GIF 使用 `palettegen` 和 `paletteuse` 两阶段生成，以控制色带和抖动质量。
- 视频引擎和工作台首次进入视频模式时懒加载；FFmpeg 核心仍由当前站点本地提供。
- 视频处理期间锁定文件、工具和参数控件；取消会终止 FFmpeg，撤销本次结果并恢复可操作状态。
- 源 MP4、结果 MP4/GIF 均可预览；成功结果进入现有本地历史保留策略。
- 截图时间必须大于等于 0 且严格小于源视频时长；支持元数据清理后的 JPG 和 PNG 输出。
- 画面调整支持 0/90/180/270° 旋转，以及原始、1:1、16:9、9:16、4:3 居中裁切；编码前将输出宽高约束为偶数。
- 视频音频导出支持 MP3、M4A 和 WAV，使用第一条可用音轨并移除源文件元数据；WAV 不显示无效码率设置。
- 参数预设保存压缩、画面调整、GIF、截图格式和音频导出参数，最多 8 条；不保存源文件、时间范围或截图时间。
- `localStorage` 中的预设数据是不可信输入，读取时必须逐字段验证名称、枚举和数字边界。
- 成功结果统一显示源文件和结果对比；结果根据 MIME 使用视频、图片或音频预览控件。

## Security Model

- 原始文件只存在于当前会话；成功结果可按本地历史规则写入当前浏览器，但不产生上传请求。
- 用户文件是不可信输入，进入转换前必须校验 MIME、类型、数量和大小。
- 文件名只通过 `textContent` 写入页面，不使用 HTML 注入。
- 选择文件、结果文件和 FFmpeg 核心的对象 URL 必须在替换、清空或离开页面时释放。
- FFmpeg 虚拟文件系统中的输入和输出在每次任务后删除。
- 音频输出移除源文件元数据，避免无意保留作者或设备信息。
- 转换期间锁定会破坏任务状态的交互。
- 取消任务后必须忽略已失效任务的迟到结果；FFmpeg 任务需要终止运行时并允许重新加载。
- ZIP 内文件名必须移除路径分隔符、控制字符和危险的前导点。
- 转换记录仅写入当前浏览器 IndexedDB，最多保留 10 条且总输出大小不超过 100 MB。
- 历史记录与下载使用的对象 URL 必须在刷新记录、清空或离开页面时释放。
- PDF 输入必须校验 MIME、数量、单文件大小、批量总大小和处理后页数。
- 新创建、合并和拆分的 PDF 不继承源文档标题、作者、主题或关键字元数据。
- PDF.js 加载任务、页面渲染任务和 Worker 必须在取消或页面销毁时终止。
- 页面管理必须限制源文件页数、复制后的输出页数、渲染像素和水印长度，避免浏览器内存被无界消耗。
- 视频输入必须限制 MIME、容器签名、文件大小、源时长和 GIF 片段时长；FFmpeg 参数只由受控枚举和已验证数值生成。
- 图片水印、批量命名和音频编辑参数必须在进入 Canvas、文件名或 FFmpeg 命令前限制长度、枚举和数值范围。
- Service Worker 不缓存非 GET、跨域或不成功的响应，离线能力不得扩大网络访问范围。

## Testing Strategy

- 单元测试覆盖文件分类、媒体 MIME 映射、输入扩展名、批量边界、输出命名、图片尺寸、质量参数、六种音频命令和整体进度。
- 单元测试覆盖 ZIP 解包与重名/路径安全、尺寸变化计算、历史数量/容量保留和取消错误分类。
- 浏览器验证覆盖图片批量转换、真实 WAV 的六种音频输出、MP3 再转 WAV、结果播放与下载、参数写入和错误恢复。
- 浏览器验证覆盖真实图片生成 PDF、PDF 转 JPG/PNG、合并顺序、页码提取、逐页拆分、ZIP 和取消。
- 浏览器验证覆盖缩略图拖拽与键盘等价操作、旋转/复制/删除、中文水印、页码、结构保留导出和实际压缩率。
- 浏览器验证覆盖真实 MP4 的压缩、时间裁剪和 GIF 双阶段生成，并检查结果可播放/预览、下载命名、本地历史和取消恢复。
- 浏览器验证覆盖真实 MP4 截图、旋转裁切、静音视频、MP3/M4A/WAV 音频导出、预设保存/应用/删除和源结果对比。
- 浏览器验证覆盖真实音频裁剪、音量、标准化、变速和淡入淡出，以及图片旋转、翻转、水印和批量重命名。
- PWA 验证覆盖清单、Service Worker 激活、同源资源缓存、断网导航回退和安装入口状态。
- 在 320、768、1024 和 1440 像素宽度检查溢出、控件尺寸、键盘顺序和无障碍树。
- 页面控制台必须没有错误或警告。

## Success Criteria

- JPG、PNG、WebP 可以单张或批量转换，也可以同格式压缩或缩放。
- 图片质量和最大尺寸会实际影响输出编码与像素尺寸。
- MP4、MP3、WAV、M4A、AAC、FLAC 和 OGG 可以转换为六种可播放、可下载的音频格式。
- 码率、采样率和声道设置会实际写入输出，WAV/FLAC 不显示无效的码率选项。
- 批量图片转换中单个文件失败不会丢失其他成功结果。
- 多个成功结果可打包为有效 ZIP，结果显示源文件大小、输出大小和变化比例。
- 取消、失败重试和刷新后历史记录均可正常工作，历史记录可下载、删除和清空。
- 选择、队列、格式、参数、进度、预览、下载和重置流程完整。
- 四种 PDF 工具均可生成可重新读取的真实结果，PDF 页面结果有预览，多文件结果可打包下载。
- 页面管理导出结果必须保持编辑后的页序与旋转角度；压缩结果不得比同次生成的结构保留版本更大。
- MP4 压缩和裁剪必须生成可播放的视频；GIF 必须按所选尺寸和帧率生成可预览图片。
- 视频截图必须生成可预览的 JPG/PNG；画面调整结果必须可播放且符合所选比例；三种音频结果必须可播放和下载。
- 参数预设必须能恢复所有非时间参数，损坏的浏览器存储不得进入转换命令。
- 音频编辑参数必须实际进入 FFmpeg 命令并生成可播放结果；图片变换、水印和连续命名必须反映在下载结果中。
- 文件信息不得触发上传请求；生产构建必须提供有效 Web App Manifest 和可激活的 Service Worker。
- 动态状态通过 `aria-live` 公布，所有交互可通过键盘访问。
- `pnpm test`、`pnpm typecheck` 和 `pnpm build` 全部通过。

## Out of Scope

- MP3 转 MP4
- Office、GIF 图片输入、HEIC 和 MP4 以外的视频输入格式
- 批量音视频转码
- 登录、会员、支付、后台和服务端持久化
